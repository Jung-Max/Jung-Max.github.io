WebJavascript

Javascript

2019-09-27
Java Script, Web

개요

Javascript에서는 특정 문자열에 포함된 문자열을 다른 문자열로 대체하기 위해 replace함수라는 것을 제공합니다.

결과는

revTest = “Who are wow? I’m your father.”

분명 뒤에 your 에 you가 또 존재하기 때문에 뒤에 있는 you도 wow로 대체될 것으로 생각했지만, 앞의 you만 wow로 대체되었다.

javascript는 위와 같은 방식으로 사용하면 처음 만나는 대체 문자만 대체한다.

그러면 어떻게 뒤에 나오는 you도 wow로 대체할 수 있을까?

즉 replace all 과 같은 기능을 구현하려면 어떻게 해야 할까?

간단하게는 javascript의 정규식(Regular expressions)을 사용하면 된다.

String revTest = test.replace(/you/g, “wow”); 또는

String revTest = test.replace(/you/gi, “wow”);

결과는

revTest = “Who are wow? I’m wowr father.”

Javascript의 정규식은 다음과 같은 형태로 사용한다.

replace(바꿀 문자, 바뀌어질 문자)

바꿀 문자는 다음과 같은 형태로 정규식으로 사용할 수 있다. (정규식을 사용하지 않으면 위의 예에서 보았듯이 대체 대상 문자 중에서 첫 번째 만다는 문자만 대체된다.)

/패턴/정규식옵션

패턴 : 대체할 문자를 입력

정규식옵션

  • g (global) : 첫번째 문자만이 아닌 패턴에 해당하는 모든 문자들을 검색하여 대체한다.
  • i (ignoreCase) : 대소문자 구분하지 않음.
  • m (multillineM) : 여러 줄 검색
  • s : “dotAll”; 줄 바꿈과 일치 하는 .을 허용합니다.
  • u : 유니 코드; 패턴을 일련의 유니 코드 코드 포인트로 취급
  • y : sticky; 대상 문자열에서이 정규식의 lastIndex 속성으로 표시된 인덱스에서만 일치하며 이후 인덱스에서는 일치하지 않습니다.

정규식의 특수 문자.

\

특수 문자가 아닌 문자 앞에 오는 백 슬래시는 다음 문자가 특수하며 문자 그대로 해석되지 않음을 나타냅니다.

예를 들어, 앞에 ‘'가없는 ‘b’는 일반적으로 소문자 ‘b’가있는 곳과 일치합니다.

문자는 문자 그대로 해석됩니다. 그러나 ‘\ b’시퀀스는 어떤 문자와도 일치하지 않습니다.

단어 경계를 나타냅니다. 특수 문자 앞에 오는 백 슬래시는 다음 문자가 특별하지 않으며 문자 그대로 해석되어야 함을 나타냅니다.

^

입력 시작과 일치합니다. 여러 줄 플래그가 true로 설정되면 줄 바꿈 문자 바로 다음에도 일치합니다.

예를 들어, /^A/는 “an A”의 ‘A’와 일치하지 않지만 “An E”의 ‘A’와 일치합니다.

^는 문자 집합 패턴에서 첫 번째 문자로 나타날 때 다른 의미를 갖습니다.

$

입력 끝과 일치합니다. 여러 줄 플래그가 true로 설정되면 줄 바꿈 문자 바로 앞에도 일치합니다.

예를 들어, /t$/는 “eater”의 ‘t’와 일치하지 않지만 “eat”에서는 일치합니다.

*

위 식을 0 번 이상 일치시킵니다. {0,}과 같습니다.

예를 들어, /bo*/는 “A ghost booooed”의 “boooo”과 “A bird warbled”의 “b”와 일치하지만 “A goat grunted”에는 없습니다.

+

위 식을 1 회 이상 일치시킵니다. {1,}과 같습니다.

예를 들어, /a+/는 “candy”의 ‘a’와 “caaaaaaandy”의 모든 a와 일치하지만 “cndy”에는 없습니다.

?

앞의 식을 0 번 또는 1 번 일치시킵니다. {0,1}과 같습니다.

예를 들어, /e?le?/는 “angel”의 ‘el’및 “angle”의 “le”및 “oslo”의 “l”과 일치합니다.

임의의 한정자 *, +,? 또는 {} 바로 다음에 사용되는 경우, 욕심이 많은 (가능한 한 많은 문자와 일치) 기본값과는 달리, 한정자가 욕심스럽지 않게 (가능한 한 가장 적은 문자와 일치) 만듭니다.

예를 들어, “123abc”에 /\d+/를 적용하면 “123”과 일치합니다. 그러나 동일한 문자열에 /\d+?/를 적용하면 “1”만 일치합니다.

.

(소수점)은 기본적으로 줄 바꿈 문자를 제외한 모든 단일 문자와 일치합니다.

예를 들어, /.n/은 “nay, an apple is on the tree”의 ‘an’및 ‘on’과 일치하지만 ‘nay’는 일치하지 않습니다.

s (“dotAll”) 플래그가 true로 설정되면 개행 문자와도 일치합니다.

(x)

다음 예와 같이 ‘x’와 일치하고 일치를 기억합니다. 괄호를 캡처 괄호라고합니다.

/(foo) (bar) \1 \2/ 패턴의 ‘(foo)’및 ‘(bar)’는 문자열 “foo bar foo bar”에서 처음 두 단어가 일치하고 기억됩니다.

\1과 \2는 첫 번째와 두 번째 괄호로 묶인 하위 문자열 일치-foo와 bar를 나타내며 문자열의 마지막 두 단어와 일치합니다.

\1, \2, …, \n은 정규식의 일치하는 부분에 사용됩니다.

정규 표현식의 대체 부분에서 $1, $2, …, $n 구문을 사용해야합니다 (예 : 'bar foo'.replace (/(...) (...)/, '$2 $1').

$&는 전체 일치 문자열을 의미합니다.

(?:x)

‘x’와 일치하지만 일치를 기억하지 않습니다. 괄호를 비 캡처 괄호라고하며 정규 표현식 연산자가 작업 할 하위 표현식을 정의 할 수 있습니다.

샘플 표현식 /(?: foo){1,2}/를 고려하십시오.

표현식이 /foo {1,2}/ 인 경우 {1,2} 문자는 ‘foo’의 마지막 ‘o’에만 적용됩니다.

캡처되지 않은 괄호로 {1,2}는 전체 단어 ‘foo’에 적용됩니다. 자세한 내용은 아래 괄호 사용을 참조하십시오.

x(?=y)

‘x’뒤에 ‘y’가있는 경우에만 ‘x’와 일치합니다. 이를 미리보기라고합니다.

예를 들어, /Jack(?=Sprat)/은 ‘Sprat’다음에 오는 경우에만 ‘Jack’과 일치합니다. /Jack(?=Sprat|Frost)/는 ‘Sprat’또는 ‘Frost’가 뒤에 오는 경우에만 ‘Jack’과 일치합니다.

그러나 ‘Sprat’나 ‘Frost’는 경기 결과의 일부가 아닙니다.

x(?!y)

‘x’뒤에 ‘y’가없는 경우에만 ‘x’와 일치합니다. 이를 부정 lookahead라고합니다.

예를 들어, /\d+(?!\.)/는 뒤에 소수점이없는 경우에만 숫자와 일치합니다.

정규식 /\d+(?!\.)/.exec("3.141")는 ‘141’과 일치하지만 ‘3.141’과는 일치하지 않습니다.

(?<=y)x

x 앞에 y가있는 경우에만 x와 일치합니다.이를 lookbehind라고합니다.

예를 들어, /(?<=Jack)Sprat/는 “Jack”이 앞에있는 경우에만 “Sprat”와 일치합니다.

/(?<=Jack|Tom)Sprat/는 “Jack”또는 “Tom”이 앞에 오는 경우에만 “Sprat”과 일치합니다.

그러나 “잭”이나 “톰”은 모두 경기 결과의 일부가 아닙니다.

(ES2018에 추가되었지만 Firefox에서는 아직 지원되지 않음)

(?<!y)x
